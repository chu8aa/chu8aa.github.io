st表

基本

  • 倍增
    • 任意整数均可表示成若干个 2 的次幂项的和
    • 顾名思义：成倍的增长，如果问题的状态空间特别大，一步一步的递推算法复杂度太高，可以通过倍增思想，只考察 2 的整数次幂位置，快速缩小求解范围，直到找到所要的解
  • ST表
    • 稀疏表 （Sparse Table），算法就是采用倍增的思想
    • O(nlogn) 的时间构造一个二维表之后，可以在O(1)的时间在线查询区间[l, r]的最值
    • ST算法可以有效的解决在线 RMQ 问题
    • f[i][j] 表示 区间[i, i + 2^(j) -1]的最值
    • f[i][j] = max/min(f[i][j - 1], f[i + 2^(j - 1)][j - 1])
    • 2^(k) <= x < 2^(k + 1)

  • 查询的时候发现只有一个位置的数和最后一个位置的数会被截断

线段树

基本

  • (segment tree) 是一种二叉搜索树，也是平衡二叉树。
  • 每一个节点都对应一个区间 [L, R]，叶子结点对应的区间只有一个节点 (L = R)
  • 每一个非叶子结点左孩子区间为[L, (L + R) / 2]，右孩子区间为[(L + R) / 2 + 1, R]
  • 树高为 log(n)，树上的操作大多和树高相关，更新和查询的种类很多，要灵活运用线段树，可以解决很多问题
  • RMQ(Rang Minimum/Maximum Query)
    • 点更新，修改一个元素的值
    • 区间查询，查询一个区间的最值
  • 空间大小要开四倍的空间
  • 分治算法策略
  • 线段树较树状数组用途更广，更灵活，更直观一些
  • 树状数组较线段树节省空间，代码简单易懂
  • 能用树状数组的问题一定能用线段树

树状数组

基本

  • 又名二进制索引树 (Binary indexed tree)
  • 分级管理系统
    • 成员 a[1], a[2]……
    • 管理小组 c[1], c[2]……
    • 根据下标来规定每个管理小组管理的范围
      • i 的二进制 结尾有几个连续的 0 就是从下标开始向前几个位置
      • lowbit(x) = x & (-x)
    • 直接前驱 一个
      • c[i - lowbit(i)]
      • 一个点和他的直接后继可能同一个直接前驱 5，6 的直接前驱是 4 ，6 是 5 的直接后继
      • 大部分是左兄弟
    • 直接后继 一个
      • c[i + lowbit(i)]
      • 父节点
    • 前驱 集合
      • 直接前驱 和 直接前驱的前驱
    • 后继 集合
      • 直接后继 和 直接后继的后继
    • 父节点管理的终点 = 该点位置和一个开始向后 该点的管理范围
    • 通过更新后继来更新后继的信息
    • 通过查询前驱得到不重不漏的前缀和
  • 多维树状数组
    • 就是多加一维的前驱或者后继
  • 树状数组的局限性
    • 要满足减法规则，比如区间和可以 sum[j] - sum[i - 1]
    • 不满足的如区间最大值 不能 qmax[j] - qmax[i - 1] 就要用线段树 或者 分块
    • 区间修改是 O(nlogn)
      • 可以用两个树状数组来维护一个数学公式来完成区间和的区间修改

例题

  • 每次查询到当前的点 在此之前有几个 y 值大于它，用已经查询的点数 - 比他小的前缀和

分块

基本

  • 树状数组和线段树虽然非常方便，但维护的信息必须满足信息合并特性（如区间可加，可减），如果不满足此特性则不能使用树状数组和线段树。分块算法可以维护一些线段树维护不了的东西，分块算法就是优化过后的暴力，它可以完成几乎所有区间更新和区间查询问题，但是效率相对于线段树等数据结构要差一些。
  • 分块算法就是将所有数据分成若干块，维护块内信息，使得块内的查询是O(1)的时间，而总的查询时间就可以看成若干块查询的总和加上两端的暴力
  • 分块算法将长度为 n 的序列分成若干块，每一块有 k 个元素，最后一块可能少于 k 个元素，为了使时间复杂度均摊，通常将块的大小设为 k = sqrt(n),用 pos[i] 表示第 i 个位置所属的块号，对于每个块都进行信息维护
  • 不要忘记可以使用 lazy 标记

默认配置的修改

修改版本号

使用 Gradle 的构建，在 AndroidManifest.xml 下 配置 versionName 会被 app 下的 build.gradle android{defaultConfig} 覆盖

修改App图标

AndroidManifest.xml 下 中 android:icon 是方形 android:roundIcon 是圆形，看系统的选择是圆形还是方形。现在图标一般是放在mipmap 下 如果不同的分辨率分开来放。在不同的机型会根据分辨率选择图标

修改App名

添加一个新的 Activity

  2. res.layout 下 布局界面 activity_main
  3. 要在 AndroidManifest.xml 下 里 配置一个
    1. 如果是启动界面要配置一些代码
    2. 如果有多个Activity 被设置为启动界面，那么默认选择第一个
布局文件的初体验
布局格式
  1. 暂时使用线性布局 LinearLayout
  • android:gravity=”center”

最小生成树

算法
Prim
  1. 初始化全部的距离为INF
  2. 遍历所有点
    1. 找到距离集合最近的点 u
    2. 如果找的这个点的距离是 INF 那么返回不存在最小树
    3. 标记这个点更新到集合中
    4. 通过这个点更新能更新的点 到集合的最短距离 dist[i] = min(dis[i], g[u][i]) 这里是和 Dijkstra 不同的地方
堆优化的Prim

和 Dijkstra 的优化操作一样。但是没啥必要。稠密图用 Prim ，稀疏图用 Kruskal

Kruskal
  1. 把所有的边按照从小到大的顺序排序
  2. 遍历排序好的边
    1. 如果两个点不联通
      1. 把当前的边加入到最小树中
      2. 把两个点标记为联通（并查集
  3. 如果边数 小于 点数 - 1 那么代表不存在最小树

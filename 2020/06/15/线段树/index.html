线段树

  1. 基本
  2. 例题
    1. HDU 1166
    2. POJ 3468
    3. HDU 4902
    4. POJ 2777

基本

  • (segment tree) 是一种二叉搜索树，也是平衡二叉树。
  • 每一个节点都对应一个区间 [L, R]，叶子结点对应的区间只有一个节点 (L = R)
  • 每一个非叶子结点左孩子区间为[L, (L + R) / 2]，右孩子区间为[(L + R) / 2 + 1, R]
  • 树高为 log(n)，树上的操作大多和树高相关，更新和查询的种类很多，要灵活运用线段树，可以解决很多问题
  • RMQ(Rang Minimum/Maximum Query)
    • 点更新，修改一个元素的值
    • 区间查询，查询一个区间的最值
  • 空间大小要开四倍的空间
  • 分治算法策略
  • 线段树较树状数组用途更广，更灵活，更直观一些
  • 树状数组较线段树节省空间，代码简单易懂
  • 能用树状数组的问题一定能用线段树

例题

HDU 1166
  • 操作
    • u 点增加或者减少 k 个人
    • [l, r] 总共多少人
  • 单点修改，区间查询
POJ 3468
  • 操作
    • [l, r] 每个点增加 k
    • [l, r] 的总和
  • 区间修改，区间查询 加一个 lazy 标记
HDU 4902
  • 操作
    1. [l, r] 每个树都修改为 x
    2. [l, r] 中每个大于 x 的数 与 x 进行 gcd 操作
    3. 查询每个点的数值
  • 操作 1. lazy 标记一下
  • 操作 2. 可以用一个 mx 来记录区间的最大值来优化，如区间最大值都小于等于 x 那么就不要进入该区间
  • 操作 3. 每个点进行查询 或者 build() 操作那样打印，时间差不多
POJ 2777
  • 操作
    • [l, r] 每个点染成 k 色
    • [l, r] 一共几个不同的颜色
  • 其实不用把数据都记录在区间中，只要记录重要的，有些可以通过查询得到的就不用在节点中开辟一段记录空间
  • 每次查询用一个 ans[] 数组来记录[l, r]中查询到的颜色
  • 要灵活使用 lazy 有一些可以直接全部区间进行标记。
  • 查询也不一定要按照一般左边右边，有时候可以根据数据去决定如何查询

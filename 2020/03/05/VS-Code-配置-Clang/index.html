VS-Code-配置-Clang

  1. 前期准备
  2. 配置环境变量
  3. 配置VSCode

官方文档

前期准备
  • 安装 Visual Studio Code
  • 安装 C++ extension for VS Code.
配置环境变量
  • 命令面板 Shell Command: Install ‘code’ command in PATH.
配置VSCode
  • 创建编译环境
    • 命令面板 C/C++:Edit Configurations (UI) 配置一下 会生成 .vscode/c_cpp_properties.json
    • Compiler path 一般走命令面板会帮你配置好
    • Set IntelliSense mode to ${default}, which on macOS is clang-x64.
    • You only need to modify the Include path setting if your program includes header files that are not in your workspace or in the standard library path. 所以这个一般不动
    • On macOS, you must set the macFrameworkPath to point to the system header files. 这个一定要设置 ，macOS 一般在这”/Library/Developer/CommandLineTools/SDKs/MacOSX.sdk/System/Library/Frameworks”

  • 配置构建任务

    • 命令面板 Create tasks.json file from template,选择Create tasks.json file from template选择Others生成.vscode/tasks.json

    • 新加的 panel 是因为出现端口占用

  • 配置调试设置

    • From the Command Palette, type “launch” and then choose Debug: Open launch.json. Next, choose the GDB/LLDB environment.

    • For program, use the program name ${workspaceFolder}/helloworld.out (which matches what you specified in tasks.json).

    • stopAtEntry = true 表示在开头设置了一个断点

    • Set externalConsole to true to display the program output in an external Terminal window. (Currently on Mac the output cannot be directed to the integrated Terminal window.)
